The Payment Destination window

Provisional in-wallet help

This window allows you to generate a list of fresh, unused payment destinations for the selected account. These destinations can be used for creating multi-output transactions (for example, splitting funds across many outputs).

In some cases these will be standard Bitcoin addresses, and in other cases where the script type has no address form, the BIP276 standard will be used to present them.

How to use

  • How many: Choose the number of destinations (outputs) to generate.
  • Mode: Select how amounts are assigned to each destination:
    • Addresses only: Generate destinations without amounts.
    • Equal split (total): Enter a total amount which will be divided evenly across all destinations.
    • Fixed amount (per address): Enter an amount that will be sent to each destination.
    • Random split (total): Enter a total amount which will be distributed randomly across destinations.
  • Amount: The meaning of this field depends on the selected mode (total vs per-address).

Tips

  • Each output has a minimum value of approximately 300 satoshis. Values below this may not be accepted or may be uneconomical to spend.
  • You can use the Pay-to-Many feature to send funds to these destinations, including sending to your own wallet to split UTXOs.
  • Large numbers of outputs may result in large transactions, which can take longer to confirm or be rejected by some servers.
  • Very small outputs may be considered uneconomical to spend later.
  • This tool can be useful for splitting UTXOs or preparing funds for distribution.

Possible dangers

The destinations listed here are not reserved. If you take them and use them elsewhere, there is no guarantee that ElectrumSV won't also proceed to use them in the course of its normal wallet usage on your behalf.
