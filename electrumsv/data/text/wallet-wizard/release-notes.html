ElectrumSVP 0.1.0: Initial stable release.

This marks the first official release of ElectrumSVP, evolving from the beta into a more stable, refined, and feature-complete wallet. This version focuses on improved transaction safety, deterministic behavior, and enhanced control over coins and fees.

For a full overview and usage guide, please refer to the official release article .

  • BREAD verification format: Replaced the previous SPV BEEF verify implementation from the beta version, with the new BREAD format (Blockchain Referenced Evidence and Data) for transaction and UTXO handling. This improves consistency, simplifies validation, and lays the foundation for future enhancements. BREAD is a subset of BEEF, but simpler and only deals with verifying funds, rather than tx ancestry.

  • Fee system overhaul: Standardized fee handling to satoshis per kilobyte (sats/KB) for predictable and transparent transaction costs. Fixed multiple edge-case bugs that could result in incorrect fee calculations or unintended 0 fees. A 1 sat minimum fee per transaction is now enforced.

  • Fee safety guard rails: Added protections to help prevent accidental overpayment of transaction fees, including improved validation and warning thresholds.

  • UTXO splitting (pay-to-many): Added advanced coin splitting tool under Account menu > Generate address batch, including:

    • Fixed-amount splits
    • Randomized output splitting for improved privacy
    • Enhanced pay-to-many workflow
    These tools allow users to prepare and manage coins more effectively, currently a 1000 utxo limit per tx, and 300 sat minimum per output. Keep in mind many outputs may slow down BREAD verification.

  • Sweep private key: Improved sweep functionality for importing funds from private keys. Supports Supports legacy (P2PKH) for compressed and uncompressed keys and BIP38 encryption, as well as BEEF/BREAD utxo import, and automatic transaction creation and broadcast.

  • Coin control & UTXO management: Improved manual coin selection and management tools. Users can "send from" certain utxos as well as freeze/unfreeze addresses and UTXOs, enabling better control over spending and privacy.

  • Destinations tab improvements: Improved handling of large scripts and output display. Better usability when working with multiple outputs and complex transactions.

  • Hardware wallet support removed: Hardware wallet functionality has been deprecated and removed in this release. This simplifies the codebase and reduces potential sources of instability. Former HW wallet users are encouraged to use a multisig setup for more security.

  • Block headers & networking: Continued improvements to header syncing and reliability. Updated server handling for more consistent connections.

  • User interface improvements:

    • General UI cleanup and layout improvements
    • Improved visibility for transaction outputs and amounts
    • More consistent behavior across wallet tabs

  • AppImage & portability: Continued improvements to Linux AppImage builds for portability and compatibility. Fully self-contained builds remain compatible with environments such as TAILS.

  • Bug fixes: Numerous fixes across transaction creation, fee calculation, UI behavior, and edge-case handling.

This release represents a major step forward in stability and usability. Further improvements will continue to focus on reliability, transparency, and user control.
